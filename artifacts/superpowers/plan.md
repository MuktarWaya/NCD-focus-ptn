# แผนงาน (Implementation Plan): เปิดใช้งานระบบความปลอดภัยรหัสผ่าน (Passcode Gate)

### Goal
เปิดใช้งานหน้าจอเข้าสู่ระบบด้วยรหัสผ่าน (Passcode Screen) เมื่อเปิดเว็บแอปพลิเคชัน เพื่อจำกัดสิทธิ์การเข้าถึงข้อมูลสุขภาพของผู้รับบริการ โดยจะตรวจสอบความถูกต้องของรหัสผ่านที่ป้อนเทียบกับรหัสผ่านที่ตั้งค่าไว้ในระบบ (เริ่มต้นคือ `123456` หรือรหัสผ่านเชื่อมโยง API ที่บันทึกใน LocalStorage)

- **การเปลี่ยนแปลง**:
  - สร้างฟังก์ชัน `checkAuthentication()` เพื่อตรวจสอบสถานะการล็อกอินใน `sessionStorage` (เช่น `ncd_authenticated`) เมื่อเปิดหน้าจอ
  - หากยังไม่ได้ล็อกอิน ให้หน้าจอ `#login-screen` แสดงผลอยู่ และขัดขวางการดึงข้อมูล/เรนเดอร์ชั่วคราว

  - ตรวจสอบรหัสผ่านที่กรอกใน `#login-passcode` เทียบกับ `state.apiPasscode` (ซึ่งบันทึกจาก LocalStorage)
  - หากถูกต้อง ให้ตั้งค่า `ncd_authenticated = 'true'` ใน `sessionStorage` แล้วซ่อน `#login-screen` พร้อมโหลด/รีเฟรชข้อมูล
  - หากไม่ถูกต้อง ให้แสดง `#login-error` สีแดงเตือน

### Verification Plan

#### Manual Verification
1. เปิดหน้าเว็บแอปพลิเคชัน และยืนยันว่าหน้าจอหลักถูกบดบังด้วยหน้าต่างล็อกอิน "ระบบติดตามพฤทีพรรมสุขภาพ"
2. ทดลองกรอกรหัสผ่านที่ผิด (เช่น `wrongpass`) และกดเข้าสู่ระบบ ยืนยันว่าหน้าจอแจ้งรหัสผ่านไม่ถูกต้องแสดงขึ้น และไม่แสดงข้อมูลหน้าเว็บ
3. กรอกรหัสผ่านที่ถูกต้อง (`123456`) และกดเข้าสู่ระบบ ยืนยันว่าหน้าจอเข้าสู่ระบบถูกซ่อน และข้อมูลแดชบอร์ดโหลดขึ้นมาตามปกติ
4. ทำการรีเฟรชหน้าเว็บในแท็บเดิม ยืนยันว่าหน้าจอล็อกอินไม่ปรากฏขึ้นอีกครั้งเนื่องจากมีการจำสถานะใน `sessionStorage` แล้ว
5. ทดลองปิดแท็บและเปิดใหม่ หรือเปิดในโหมดไม่ระบุตัวตน (Incognito) ยืนยันว่าจะต้องทำการล็อกอินใหม่อีกครั้ง
