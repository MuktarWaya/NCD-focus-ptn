# แผนงาน (Implementation Plan)

### Goal
ปรับปรุงเว็บแอปพลิเคชันส่วนหน้าบ้าน (`index.html`, `style.css`, `app.js`) ให้ใช้เฟรมเวิร์ก Tailwind CSS และมีหน้าตาสไตล์ Glassmorphism ตรงตามงานออกแบบจาก Google Stitch พร้อมกับเพิ่มฟังก์ชันเครื่องมือแปลผลและบันทึกผลการชั่งน้ำหนักองค์ประกอบร่างกาย (Body Composition Interpreter) สำหรับ อสม. และคณะกรรมการหมู่บ้าน โดยคงฟังก์ชันเชื่อมโยง Google Sheets API (GAS API) และฟังก์ชันเดิมครบถ้วน

### Plan Steps

1. **ปรับปรุงโครงสร้าง HTML5 ให้เป็นดีไซน์ Tailwind และเพิ่มมุมมองเครื่องมือแปลผล**
   - Files: [MODIFY] `index.html`
   - Change: 
     - นำเข้า Tailwind CSS และกำหนดค่า config สีกำหนดเองจาก Stitch (เช่น `primary`, `background-base`, `surface-glass`, `risk-danger`)
     - ปรับโครงสร้าง Sidebar, Header, และ View Sections (Dashboard, Targets, Profile, Settings)
     - เพิ่มปุ่มเมนูนำทางใน Sidebar สำหรับ "เครื่องมือแปลผลร่างกาย (Body Composition Interpreter)"
     - เพิ่มหน้าจอ View ใหม่ (`#interpreter-view`) ซึ่งประกอบด้วยฟอร์มป้อนข้อมูล (เพศ, ส่วนสูง, น้ำหนัก, %ไขมัน, ไขมันช่องท้อง, %กล้ามเนื้อลาย, อายุร่างกาย) และการ์ดแสดงผลลัพธ์การแปลผลแบบ Real-time พร้อมแถบสีความเสี่ยงและการให้คำแนะนำเบื้องต้น
     - ปรับปรุงฟอร์มบันทึกผล 3 เดือน (`#quarterly-modal`) ให้มีส่วนแสดงผลการแปลผล (Badge สถานะ) ข้างๆ ช่องอินพุตข้อมูลที่เกี่ยวข้อง เพื่อนำทาง อสม. แบบเรียลไทม์
   - Verify: เปิดดูความถูกต้องของอิลิเมนต์ HTML5 และตรวจสอบว่าไม่มี ID เดิมตัวใดตกหล่นไป

2. **ปรับสไตล์ CSS เพื่อรองรับเอฟเฟกต์ Glassmorphism**
   - Files: [MODIFY] `style.css`
   - Change: 
     - ลดขนาดไฟล์และลบโค้ดสไตล์เก่าที่ใช้ Vanilla CSS ออก (เนื่องจากย้ายไปใช้ Tailwind แล้ว)
     - เขียน CSS ใหม่เฉพาะสไตล์ตกแต่งระดับไมโคร เช่น คลาสกระจกแก้วฝ้าโปร่งแสง (`.glass-card`, `.glass-floating`), การเบลอพื้นหลัง และคลาสกำหนดขอบสีบอกระดับความเสี่ยงของเป้าหมาย
   - Verify: ตรวจทานในเบราว์เซอร์ให้เห็นพื้นหลังไล่เฉดและการ์ดโปร่งใสลอยตัวสวยงาม

3. **เขียนระบบตรรกะการแปลผล (Interpretation Engine) และควบคุมหน้าจอใน JavaScript**
   - Files: [MODIFY] `app.js`
   - Change:
     - พัฒนาฟังก์ชันตรรกะแปลผล (Interpretation rules) สำหรับ BMI, เปอร์เซ็นต์ไขมันในร่างกาย (แยกเพศ), ไขมันช่องท้อง, เปอร์เซ็นต์กล้ามเนื้อลาย (แยกเพศ) อ้างอิงตามเกณฑ์รูปภาพของ OMRON / WHO / Lohman
     - เชื่อมโยงตัวป้อนข้อมูลในหน้าเครื่องมือแปลผล (`#interpreter-view`) ให้ทำงานแบบ Real-time (ทันทีที่กรอกหรือเปลี่ยนค่า จะคำนวณและแปลผลแสดงผลลัพธ์บนการ์ดอย่างสวยงาม)
     - เชื่อมโยงตัวป้อนในฟอร์มบันทึก 3 เดือน ให้คำนวณ BMI อัตโนมัติ และแสดง Badge แปลผลของแต่ละค่าเคียงข้างช่องป้อนข้อมูล
     - ปรับปรุงฟังก์ชันเรนเดอร์ไทม์ไลน์ 3 เดือน (`renderQuarterlyTimeline`) ให้แสดง Badge แปลผลสรุปสถานะสุขภาพ (เช่น ไขมัน: ปกติ, กล้ามเนื้อ: ต่ำ)
     - อัปเดตฟังก์ชันเชื่อมโยงหน้าจอ (Sidebar navigation) เพื่อรองรับแท็บเครื่องมือแปลผล
     - ปรับแต่งสีสันและเส้นกราฟของ Chart.js ให้สอดคล้องกับจานสีหลักและสีรองของ Stitch
   - Verify: รันโค้ดและดีบักระบบคำนวณ ตรวจสอบกรณีทดสอบต่างเพศ/ต่างช่วงน้ำหนักว่าได้ค่าสถานะความเสี่ยง (ปกติ/สูง/ต่ำ) ตรงตามตารางอย่างถูกต้อง

---

### Risks & Mitigations
1. **อสม. สับสนหรือคีย์ข้อมูลผิดพลาด**:
   - *Mitigation*: ระบบจะทำการประมวลผลทันทีและแสดง Badge ข้อความพร้อมสีสถานะที่เด่นชัด (เขียว = ปกติ, แดง = สูงมาก/เสี่ยง, ส้ม = สูง, ฟ้า/เทา = ต่ำ) ทำให้ อสม. ทราบทันทีหากค่าผิดปกติ และช่วยกรอก BMI จากน้ำหนักและส่วนสูงโดยอัตโนมัติเพื่อลดความยุ่งยาก
2. **การรบกวนฐานข้อมูล Google Sheets เดิม**:
   - *Mitigation*: ระบบการแปลผลและการคำนวณจะถูกจัดการบนฝั่ง Client (เบราว์เซอร์) ทั้งหมด และข้อมูลที่ส่งบันทึกใน Google Sheets จะยังคงเป็นตัวเลขดิบตามคอลัมน์มาตรฐานเดิม ไม่รบกวนโครงสร้างแถวและคอลัมน์ใน Sheets
