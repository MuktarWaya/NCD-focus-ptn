# เอกสารสรุปการส่งมอบงานโครงการ (Handover & Migration Notes)
**ระบบติดตามพฤติกรรมสุขภาพ "มุ่งเป้าปัตตานี" (NCDs Focus App)**

เอกสารฉบับนี้จัดทำขึ้นเพื่อสรุปการออกแบบ โครงสร้างการพัฒนา ปัญหาที่พบและการแก้ไข รวมถึงแนวทางการดำเนินการต่อเมื่อคุณย้ายไปใช้เครื่องมือพัฒนาอื่น (เช่น Codex)

---

## 1. ข้อมูลทางสถาปัตยกรรมเทคโนโลยี (Technical Architecture)

- **Frontend (หน้าบ้าน):** 
  - เป็นแอปพลิเคชันหน้าเดียว (Single Page Application - SPA) พัฒนาด้วย HTML5 และ JavaScript ดั้งเดิม (Vanilla JS)
  - **การตกแต่งสไตล์ (CSS):** ใช้เฟรมเวิร์ก **Tailwind CSS CDN** ผสมผสานกับการเขียนตกแต่งใน `style.css` เพื่อเน้นเอฟเฟกต์ **Glassmorphism** (การ์ดฝ้ากระจกโปร่งแสง เบลอหลัง) และรองรับ **Dark Mode** ตามสไตล์การออกแบบของ Google Stitch
  - **การแสดงผลกราฟ:** ใช้ไลบรารี **Chart.js** วาดกราฟเส้นเพื่อวิเคราะห์แนวโน้มสุขภาพรายไตรมาส (น้ำหนัก/BMI, ความดันโลหิต, ค่าน้ำตาล DTX)
- **Backend (หลังบ้าน):** 
  - ทำงานบน **Google Apps Script (GAS)** ที่ผูกเข้ากับ Google Sheets เพื่อทำหน้าที่เป็นฐานข้อมูลเก็บประวัติ
  - แลกเปลี่ยนข้อมูลด้วยมาตรฐาน REST API ผ่านคำขอ POST (ส่ง payload แบบ Plain Text เพื่อหลีกเลี่ยงข้อจำกัด CORS Preflight ของเบราว์เซอร์)
- **โหมดทำงาน (Working Modes):**
  - **Online Mode:** เชื่อมโยงอ่าน/เขียนข้อมูลจริงกับ Google Sheets ผ่าน Web App URL
  - **Demo Mode:** โหมดทดสอบออฟไลน์จำลองการทำงานโดยดึงข้อมูลจากไฟล์ CSV ท้องถิ่นในระบบเก็บไว้บนหน่วยความจำเบราว์เซอร์ (สำหรับทดลองเล่นโดยไม่ต้องเปิดใช้ชีต)

## 2. ฟีเจอร์สำคัญที่ติดตั้งล่าสุด (Key Features Implemented)

### 🎨 2.1 ดีไซน์สไตล์ Modern Glassmorphism (Google Stitch)
- ปรับโครงสร้างเมนู Sidebar ให้นำทางง่ายด้วย Material Symbols Outlined
- แปลงบอร์ดรายการกลุ่มเป้าหมาย (Targets List) เป็น Bento-card ที่แสดงการประเมินผลระดับดัชนีสุขภาพ (น้ำหนัก, น้ำตาล, ความดัน) เป็นแถบสีสถานะความเสี่ยง (เขียว = ปกติ, เหลือง = เฝ้าระวัง, แดง = วิกฤต/เสี่ยงสูง)
- หน้าจอตอบสนองเหมาะสม (Responsive Layout) ทั้งหน้าจอคอมพิวเตอร์และแท็บเล็ต/มือถือ อสม.

### ⚖️ 2.2 เครื่องมือแปลผลองค์ประกอบร่างกาย (Body Composition Interpreter)
- ติดตั้งเกณฑ์แปลผลสุขภาพมาตรฐานอ้างอิงคู่มือเครื่องชั่ง OMRON / WHO / Lohman แยกประเภทการประเมินเพศชายและหญิง ได้แก่:
  - **ดัชนีมวลกาย (BMI):** ต่ำกว่าเกณฑ์, ปกติ, น้ำหนักเกิน, อ้วน
  - **เปอร์เซ็นต์ไขมันในร่างกาย (% Body Fat):** ต่ำ, ปกติ, สูง, สูงมาก (แยกเพศ)
  - **ระดับไขมันช่องท้อง (Visceral Fat Level):** ปกติ (1-9), สูง (10-14), สูงมาก (15-30)
  - **เปอร์เซ็นต์กล้ามเนื้อลาย (% Skeletal Muscle):** ต่ำ, ปกติ, สูง, สูงมาก (แยกเพศ)
- ระบบแปลผลประมวลผลทันทีขณะพิมพ์ตัวเลขดิบลงในหน้าเครื่องมือแปลผลแบบด่วน (`#interpreter-view`) และมีปุ่มนำข้อมูลไปกรอกลงฟอร์ม 3 เดือนทันที
- ติดตั้งระบบตรวจจับเดาเพศอัตโนมัติจากคำนำหน้าชื่อ (`inferGender` จาก นาย/ด.ช. -> ชาย และ นาง/นางสาว/ด.ญ. -> หญิง)

### 🔒 2.3 ระบบความปลอดภัยรหัสผ่าน (Passcode Gate)
- เมื่อเข้าหน้าเว็บแอป หน้าจอล็อกอินเต็มจอ (`#login-screen`) จะบล็อกไม่ให้เข้าถึงข้อมูลทั้งหมดและยับยั้งการยิงคำขอ API ไปที่ Google Sheets เพื่อความปลอดภัยของข้อมูล
- รหัส demo สำหรับเข้าใช้งานหน้าเว็บเมื่อยังไม่ได้ตั้งค่า API คือ **`123456`** ส่วนรหัสผ่าน API จริงต้องตั้งใน Apps Script Script properties (`API_PASSCODE`) และไม่ควร commit ลง GitHub
- จดจำสถานะล็อกอินผ่าน `sessionStorage` เพื่อให้ไม่เกิดหน้าจอรบกวนรหัสผ่านอีกรอบเมื่อโหลดหน้าใหม่ในแท็บเดิม

## 3. ปัญหาที่พบระหว่างการเชื่อมต่อและการแก้ไขสำเร็จ (Resolved Issues)

ระหว่างการประกอบหน้าบ้านเชื่อมกับ Google Sheets ของคุณ ได้เจอปัญหาเชิงเทคนิคและได้รับการแก้ไขให้เสถียร 100% แล้วดังนี้:

| ปัญหาที่ตรวจพบ | สาเหตุหลัก | วิธีแก้ปัญหาที่ดำเนินการแล้ว |
| :--- | :--- | :--- |
| **1. ฐานข้อมูล/สคริปต์ว่างเปล่า** | ระบบ Google Clasp ติดข้อจำกัดล็อกอิน OAuth บัญชีของคุณจากเครื่องเบื้องหลังของผม | เปลี่ยนไปใช้วิธีนำเสนอโค้ดใน [gas_backend.js](file:///g:/My%20Drive/%E0%B8%AA%E0%B8%98%E0%B8%99.%E0%B8%A1%E0%B8%B2%E0%B8%A2%E0%B8%AD/%E0%B8%81%E0%B8%A5%E0%B8%B8%E0%B9%88%E0%B8%A1%E0%B8%87%E0%B8%B2%E0%B8%99%E0%B8%9E%E0%B8%B1%E0%B8%92%E0%B8%99%E0%B8%B2%E0%B8%9A%E0%B8%A3%E0%B8%B4%E0%B8%81%E0%B8%B2%E0%B8%A3%E0%B9%81%E0%B8%A5%E0%B8%B0%E0%B8%A7%E0%B8%B4%E0%B8%8A%E0%B8%B2%E0%B8%81%E0%B8%B2%E0%B8%A3/webapp/%E0%B8%A3%E0%B8%B0%E0%B8%9A%E0%B8%9A%E0%B8%95%E0%B8%B4%E0%B8%94%E0%B8%95%E0%B8%B2%E0%B8%A1%E0%B8%9E%E0%B8%A4%E0%B8%95%E0%B8%B4%E0%B8%81%E0%B8%A3%E0%B8%A3%E0%B8%A1%20%E0%B8%A1%E0%B8%B8%E0%B9%88%E0%B8%87%E0%B9%80%E0%B8%9B%E0%B9%89%E0%B8%B2%E0%B8%9B%E0%B8%B1%E0%B8%95%E0%B8%95%E0%B8%B2%E0%B8%99%E0%B8%B5/backend/gas_backend.js) ให้คุณคัดลอกไปวางด้วยตัวเอง |
| **2. เออร์เรอร์ `dateStr.split is not a function`** | Google Sheets ส่งช่องวันที่ออกมาในรูปแบบวัตถุ Date (ไม่ใช่ข้อความ String) การ split วันที่จึงล้มเหลว | ปรับปรุงโค้ดหลังบ้านให้เช็ค `instanceof Date` หากเป็นวัตถุวันที่ให้ส่งผ่านได้ทันทีโดยไม่ต้องทำการตัดคำ |
| **3. ปุ่มแก้ไขและบันทึกไม่ตอบสนอง** | คอลัมน์ความดันโลหิต (`bp`) บางแถวถูกเก็บเป็นข้อมูลชนิดตัวเลข (เช่น 120) ทำให้คำสั่ง `.split('/')` หน้าบ้านแครชและ JavaScript ค้าง | ปรับปรุงโค้ดฝั่งหน้าบ้านทั้งหมดใน `app.js` และหลังบ้าน ให้แปลงความดันเป็นข้อความด้วย `.toString()` เสมอก่อนนำไปแยกค่าบนตัวแปร |
| **4. คีย์ข้อมูลบุคคลใหม่ไม่บันทึก** | Google Sheets ของคุณว่างเปล่า ไม่มีชื่อคอลัมน์หัวตาราง (Headers) ในแถวที่ 1 ระบบสคริปต์จึงเขียนข้อมูลลงเซลล์ไม่ได้ | เขียนฟังก์ชันพิเศษ **`setupSheets`** ในหลังบ้านเพื่อให้รันสร้างชีตและเขียนหัวตาราง Row 1 ให้อัตโนมัติ |

## 4. แผนงานขั้นต่อไปเมื่อย้ายสิทธิ์การพัฒนาไปยัง Codex (Next Steps)

เมื่อย้ายการคู่โค้ดไปยัง Codex คุณสามารถดำเนินงานต่อได้ทันทีดังนี้:

### ขั้นตอนที่ 4.1: การติดตั้งฐานข้อมูล Google Sheets และอัปเดต API (ฝั่งหลังบ้าน)
1. เปิดหน้าสคริปต์ Apps Script Editor ของโปรเจกต์ที่ผูกกับ Google Sheet ฐานข้อมูลของคุณ
2. วางโค้ดล่าสุดจากไฟล์ [gas_backend.js](file:///g:/My%20Drive/%E0%B8%AA%E0%B8%98%E0%B8%99.%E0%B8%A1%E0%B8%B2%E0%B8%A2%E0%B8%AD/%E0%B8%81%E0%B8%A5%E0%B8%B8%E0%B9%88%E0%B8%A1%E0%B8%87%E0%B8%B2%E0%B8%99%E0%B8%9E%E0%B8%B1%E0%B8%92%E0%B8%99%E0%B8%B2%E0%B8%9A%E0%B8%A3%E0%B8%B4%E0%B8%81%E0%B8%B2%E0%B8%A3%E0%B9%81%E0%B8%A5%E0%B8%B0%E0%B8%A7%E0%B8%B4%E0%B8%8A%E0%B8%B2%E0%B8%81%E0%B8%B2%E0%B8%A3/webapp/%E0%B8%A3%E0%B8%B0%E0%B8%9A%E0%B8%9A%E0%B8%95%E0%B8%B4%E0%B8%94%E0%B8%95%E0%B8%B2%E0%B8%A1%E0%B8%9E%E0%B8%A4%E0%B8%95%E0%B8%B4%E0%B8%81%E0%B8%A3%E0%B8%A3%E0%B8%A1%20%E0%B8%A1%E0%B8%B8%E0%B9%88%E0%B8%87%E0%B9%80%E0%B8%9B%E0%B9%89%E0%B8%B2%E0%B8%9B%E0%B8%B1%E0%B8%95%E0%B8%95%E0%B8%B2%E0%B8%99%E0%B8%B5/backend/gas_backend.js) ทั้งหมดลงไปแทนที่ของเดิมแล้วกดบันทึก
3. เลือกรันฟังก์ชัน **`setupSheets`** เพื่อเตรียมพร้อมคอลัมน์บน Google Sheets ของคุณ
4. ทำการ **Deploy (New deployment)** สคริปต์ให้เป็น **Web App**
   - *Execute as:* `Me`
   - *Who has access:* `Anyone`
   - คัดลอกลิงก์ Web App URL ที่ลงท้ายด้วย `/exec` มาอัปเดตลงในตัวแปร `gasUrl` เริ่มต้นในไฟล์ `app.js` ของคุณ (ซึ่งปัจจุบันได้เชื่อมกับลิงก์ของคุณสำเร็จแล้ว)

### ขั้นตอนที่ 4.2: การนำเข้าข้อมูลตัวอย่าง CSV เข้า Google Sheets
หากคุณต้องการข้อมูลผู้รับบริการจำลองทั้งเกือบสองร้อยคนสำหรับเริ่มรันในแอป:
1. เปิดแผ่นงาน Google Sheets
2. ในแท็บ `Targets` ให้กด **ไฟล์ -> นำเข้า -> อัปโหลด** แล้วเลือกไฟล์ `data/targets.csv` -> เลือกตำแหน่งนำเข้าเขียนทับแผ่นงานปัจจุบัน
3. ในแท็บ `QuarterlyData` และ `DailyLogs` ทำการนำเข้าไฟล์ `data/quarterly_data.csv` และ `data/daily_logs.csv` ตามลำดับ

### ขั้นตอนที่ 4.3: การนำหน้าบ้าน Deploy ผ่าน Netlify
- เนื่องจากโปรเจกต์ได้รับการทดสอบความถูกต้องของ Syntax และการทำงานแบบออฟไลน์และออนไลน์เสร็จสิ้นแล้ว เมื่อคุณเชื่อมโยง GitHub Repository ของคุณเข้ากับ Netlify โครงสร้างไฟล์ทั้งหมดจะได้รับการสร้างและอัปเดตออนไลน์ให้คุณใช้งานทันทีที่หน้าเบราว์เซอร์
